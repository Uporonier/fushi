
SVD 是靠“猜”用户和物品的隐藏特征，而 KNN（K-Nearest Neighbors） 则是靠“算”真实的相似度。
我们可以把 KNN 的推荐逻辑形象地理解为“找邻居”。

---

1. 基于用户的协同过滤 (User-Based CF)
    核心思想：“臭味相投”。如果用户 A 和用户 B 过去喜欢的礼物很像，那么就把 B 喜欢但 A 还没买过的礼物推荐给 A。
    推荐流程：输入：目标用户 ID。计算相似度：计算目标用户与其他所有用户之间的余弦相似度（sim_options 里的 cosine）。
    寻找邻居：找到与目标用户最相似的 K 个邻居。
    预测评分：查看这些邻居对某个“目标礼物”的评分，然后进行加权平均。
    输出：预测评分最高的前 N 个礼物。

---

2. 基于物品的协同过滤 (Item-Based CF)
核心思想：“物以类聚”。如果礼物 A 和礼物 B 经常被同一群人喜欢，那么 A 和 B 就是相似的。如果用户买过 A，就推荐相似的 B。

你的代码 get_top_n_recommendations_item_based 实际上手动实现了一个非常直观的逻辑：

代码解析：
```Python
for gift_id in unknown_gift_ids:
    # 1. 遍历用户没见过的每个礼物 (gift_id)
    for known_gift_id in known_gift_ids:
        # 2. 拿出用户已经买过的每个礼物 (known_gift_id)
        # 3. 从 algo.sim 矩阵中直接查出这两个礼物的“相似度分数”
        sim = algo_item_base.sim[inner_gift_id][inner_known_gift_id]
        sim_scores.append(sim)
    
    # 4. 求均值：如果这个新礼物和用户买过的所有礼物平均相似度很高，就推荐它
    avg_sim_score = sum(sim_scores) / len(sim_scores)
```

KNN 的向量是怎么产生的？（训练过程）
在 KNN 中，向量就是原始数据本身，不需要“创造”新特征。

初始状态：你的原始评分表就是向量。

向量是什么：

用户的向量：是他对所有礼物的评分列表。比如用户 1 的向量是 [礼物1: 5分, 礼物2: 0分, 礼物3: 2分...]。

礼物的向量：是所有用户对它的评分列表。比如礼物 A 的向量是 [用户1: 5分, 用户2: 3分, 用户3: 0分...]。

训练目标：计算并存储一个相似度矩阵（Similarity Matrix）。

训练了什么：
algo.fit(trainset) 在 KNN 里其实没干别的，它只是拿着尺子（余弦公式）两两测量这些原始向量之间的夹角，然后把结果存进一张巨大的表（algo.sim）里。

### SVD 与 KNN 核心区别对比表

| 特性 | SVD (矩阵分解) | KNN (最近邻算法) |
| :--- | :--- | :--- |
| **核心原理** | **特征建模**：将用户和物品映射到低维隐向量（特征空间）。 | **邻居搜索**：直接计算用户之间或物品之间的相似度距离。 |
| **推荐逻辑** | **预测导向**：计算用户向量与物品向量的点积来预测评分。 | **匹配导向**：寻找与目标“最像”的邻居进行推荐。 |
| **直观性** | **黑盒**：很难解释为什么推荐（无法直观看到特征含义）。 | **白盒**：非常直观（如：因为你买过 A，所以推荐相似的 B）。 |
| **计算压力** | **训练慢，预测快**：离线训练耗时，但线上推荐响应极快。 | **训练快，预测慢**：仅需存相似度矩阵，但查找邻居计算量大。 |
| **数据稀疏性** | **抗稀疏性强**：适合处理大规模、稀疏的评分数据。 | **抗稀疏性弱**：如果重合评分太少，相似度计算会失真。 |
